## 2026-07-16

- **Lexical layer: Wiktionary base + standoff override layer.** Per Brett (confidence ~0.7 over OED-keyed standoff ~0.15 and own lexicon ~0.1): the lexical end of the constructicon keys to Wiktionary via the wiktextract JSONL dumps at kaikki.org, because it's the only English dictionary simultaneously CC BY-SA licensed, machine-readable at scale, broad enough, and freely checkable by readers. The base is never vendored (~1.38M words, ~3 GB English JSONL; dump of 2026-07-06); the repo publishes only a standoff CGEL-category override layer keyed at **lemma + source POS, not sense ID** (sense IDs unstable; Kilgarriff). Method: default POS→CGEL mapping rules, hand-curated closed-class gold lists that override them, lexeme-by-lexeme adjudication of the open-class residue with distributional tests recorded as evidence, published as versioned data. Seeded in `data/lexicon/` (README, `pos-defaults.yaml`, `overrides.yaml`) + `data/schemas/lexicon-override.json`; layer licensed CC BY-SA 4.0. Gold lists to be populated only after the CGEL audit corrections land in the determinatives inventory.
- **Agent audit reports live in the repo, not /tmp.** Codex's CGEL category audit (29 findings) moved to `docs/audits/2026-07-16-cgel-category-audit.md`; /tmp outputs are useless to Brett.
- **Absolute negators are determinatives, not pronouns.** Brett caught the mislabel; CGEL Ch. 9 §3.2 confirms: no/none are dependent/independent forms of the determinative no, and nobody/no one/nothing/nowhere/no place are single-word compounds of no. Entry renamed `absolute-negator-pronoun-001` → `absolute-negator-determinative-001`; references in no-determiner, never-adv-negation, neither-nor-coordination, and the form-features index updated. Standing alone as NPs, these occupy fused determiner-head function.
- **Projectibility-first supersedes HPC framing.** Per Brett: the framework is projectibility-first (projectibility is the target of kind claims; homeostasis is the strictest rung of a diagnostic ladder, per Reynolds 2026, "Not every stable cluster is homeostatic"). CONTRIBUTING.md written in these terms.
- **Full projectibility-first conversion (same day, on Brett's correction that the deprecation was to be implemented, not just noted).** Schema block renamed `hpc` → `kind`: `projection` (target/evidence/confidence) leads; `secured` records the earned tier (`stable`/`networked`/`maintained`/`controlled`); `failure-mode` (thin/fat/negative) replaces the old status labels; `cluster`→`profile`, `homeostasis`→`maintenance`; new `network` and `control` (four marks) fields for their rungs. Migration deflated honestly: no entry keeps the old `status: hpc` overclaim; five entries with recorded projection evidence got `secured: stable` + `provisional: true`; `adjective-001` → `failure-mode: fat`; `vocative-001` and `directive-ostensive-001` lack projection targets (lint now warns). Lint enforces ladder discipline (claimed tier without rung evidence warns; `controlled` without all four control marks fails). `docs/hpc-implications.md` → `docs/projectibility-first.md`; CLAUDE.md, SCHEMA.md, GEMINI.md, STATUS.md, HANDOFF.md, kind-status index all converted.
- **Public deployment via GitHub Pages + CI.** Static browser deploys from a workflow (`pages.yml`); `validate.yml` runs schema validation + lint on every push/PR. `web/app.js` gained base-path resolution so root-absolute manifest paths work under `/<repo>/`. Untracked `literature/` (CGEL PDF excerpts, copyrighted) stays out of the repo and the site; the manifest builder now drops empty sections.
